# Шагомер — Telegram бот + дашборд для конкурса шагов

> Полное решение на Yandex Cloud: скриншоты каждого участника хранятся в отдельной папке, данные — в CSV в облаке, дашборд показывает рейтинг в реальном времени.

## Архитектура

```
┌──────────────┐      ┌──────────────────────┐      ┌─────────────┐
│  Telegram    │─────▶│  Yandex Object       │◄─────│   Web       │
│  Bot (aiogram)│     │  Storage (S3)        │      │   Dashboard  │
│              │      │                      │      │   (React)    │
│  ├─ скриншоты│─────▶│  screenshots/        │      └──────┬──────┘
│  │  в папки  │      │    ├── 123456/        │             │
│  │  юзеров   │      │    │   ├── *.jpg      │             │
│  │           │      │    └── 789012/        │             │
│  └─ данные   │─────▶│  data/               │      ┌────────┴────────┐
│     в CSV    │      │    ├── steps.csv      │      │  FastAPI        │
│              │      │    └── participants.csv│◄─────│  (REST API)     │
└──────────────┘      └──────────────────────┘      └─────────────────┘
```

## Что включено

| Компонент | Технология | Описание |
|-----------|-----------|----------|
| **Telegram-бот** | Python + aiogram | Принимает шаги, сохраняет в облако |
| **REST API** | FastAPI | Дашборд читает данные через API |
| **Веб-дашборд** | React + TypeScript | Рейтинг, графики, статистика |
| **Хранилище** | Yandex Object Storage | S3-совместимое, дешевое, надёжное |

## Особенности

- **Отдельные папки** — каждый пользователь имеет свою папку `screenshots/{user_id}/` для скриншотов
- **Проверяемые источники** — скриншоты хранятся в облаке, можно перепроверить в любой момент
- **Yandex Cloud** — всё в облаке РФ, низкая цена (~бесплатно для небольших объёмов)
- **Автообновление** — дашборд обновляется каждые 30 секунд

## Быстрый старт

### 1. Настройка Yandex Cloud

#### 1.1 Регистрация

1. Зайди на [console.yandex.cloud](https://console.yandex.cloud/)
2. Создай аккаунт (или войди через Яндекс.ID)
3. Создай платёжный аккаунт (требует карту, но первые 3 месяца — грант на 4000₽)

#### 1.2 Создай бакет (Object Storage)

1. Перейди в **Object Storage → Бакеты**
2. Нажми **Создать бакет**
3. Название: `steps-competition` (или любое уникальное)
4. Доступ: **Ограниченный**
5. Класс хранилища: **Стандартный**
6. Регион: `ru-central1`

#### 1.3 Создай сервисный аккаунт

1. Перейди в **IAM → Сервисные аккаунты**
2. Нажми **Создать сервисный аккаунт**
3. Имя: `steps-bot`
4. Нажми **Создать**

#### 1.4 Создай статический ключ доступа

1. Открой сервисный аккаунт `steps-bot`
2. Перейди во вкладку **Статические ключи доступа**
3. Нажми **Создать статический ключ**
4. Описание: `Для бота шагомера`
5. Сохрани **Access key ID** и **Secret access key** (покажутся один раз!)

#### 1.5 Дай права на бакет

1. Перейди в **Object Storage → Бакеты**
2. Открой свой бакет
3. Вкладка **Права доступа**
4. Нажми **Добавить пользователя**
5. Выбери сервисный аккаунт `steps-bot`
6. Роль: **Редактор** (`storage.editor`)

---

### 2. Настройка Telegram-бота

#### 2.1 Получи токен

1. Напиши [@BotFather](https://t.me/BotFather) в Telegram
2. Отправь `/newbot`
3. Придумай имя и username
4. Скопируй токен (выглядит как `123456789:ABCdef...`)

#### 2.2 Настрой окружение

```bash
cd bot
cp .env.example .env
```

Открой `.env` и заполни:

```env
BOT_TOKEN=твой_токен_от_BotFather
YC_ACCESS_KEY=твой_access_key
YC_SECRET_KEY=твой_secret_key
YC_BUCKET_NAME=steps-competition
API_BASE_URL=http://твой_сервер:8000   # или localhost для тестов
```

#### 2.3 Установи зависимости

```bash
# Требуется Python 3.9+
pip install -r requirements.txt
```

#### 2.4 Запусти

```bash
# Бот + API-сервер одновременно:
python main.py
```

Бот запустится в Telegram, API — на `http://localhost:8000`.

## Как пользователи отправляют шаги

### Форматы сообщений боту:

| Формат | Пример | Результат |
|--------|--------|-----------|
| Число | `8500` | Шаги за вчера |
| Дата + шаги | `15.01 8500` | Шаги за 15 января |
| С разделителем | `15.01: 8500` | Шаги за 15 января |
| Полный формат | `Дата: 15.01, Шаги: 8500` | Шаги за 15 января |
| ISO дата | `2024-01-15 8500` | Шаги за 15 января |
| Со скриншотом | Фото + `8500` в подписи | Шаги + скриншот в облаке |

### Команды бота:

| Команда | Описание |
|---------|----------|
| `/start` | Регистрация, приветствие |
| `/my_stats` | Личная статистика |
| `/leaderboard` | Текущий рейтинг |
| `/help` | Справка |

## Деплой на сервер (для 24/7 работы)

### Вариант A: Yandex Cloud (рекомендуется)

Разверни бота на **Yandex Cloud Functions** или **Compute Cloud**:

```bash
# На сервере:
apt update && apt install python3-pip -y
git clone <твой-репозиторий>
cd steps-competition/bot
pip3 install -r requirements.txt

# Запуск через systemd или screen:
screen -S bot
python3 main.py
# Ctrl+A, D — отключиться
```

### Вариант B: Railway / Render (бесплатно)

1. Загрузи код на GitHub
2. Подключи [Railway](https://railway.app) или [Render](https://render.com)
3. Укажи переменные окружения в настройках
4. Команда запуска: `python bot/main.py`

## FAQ

**Q: Где хранятся скриншоты?**
A: В Yandex Object Storage, каждый пользователь — в своей папке: `screenshots/{user_id}/`.

**Q: Можно ли перепроверить данные?**
A: Да! Скриншоты хранятся в облаке, админ может зайти в бакет и проверить.

**Q: Бот будет работать у меня на компьютере?**
A: Да, но только пока запущен. Для 24/7 нужен сервер (см. раздел "Деплой").

**Q: Можно использовать без Яндекса?**
A: Можно адаптировать `storage.py` под любое S3-совместимое хранилище (AWS S3, MinIO, Selectel).

**Q: Как добавить участника вручную?**
A: Отредактируй `data/participants.csv` в бакете через веб-интерфейс Object Storage.
